# SKILL: Summary — Classification Report Generation

## Purpose

Format the final classification result as a **human-readable Markdown report**
that can be displayed directly to the user or embedded in API responses.

## Note: Deterministic Phase — No LLM Call Required

The summary is assembled programmatically from the Act and Observe outputs.
No LLM call is made.

## Report Template

```markdown
# HS Code Classification Report

## Product Description
{product_description}

### Declaration Elements
- {element_name}: {element_value}
- ...

## HS Code Candidates

**Primary Candidate:** `{hs6_code}`

**Confidence:** {confidence}

**Ambiguity Status:** ✅ Unambiguous   (or ⚠️ Ambiguous)

**Reason:** {ambiguity_reason}          (only if is_ambiguous = true)

**Reasoning:** {reasoning}

**GIR Analysis:**
- {rule_1}
- {rule_2}

---
*Generated by SmartHSCodeAgent*
```

## Output Structure

```json
{
  "markdown": "<full markdown string as above>",
  "product_description": "...",
  "declaration_elements": [...],
  "hs6_code": "XXXXXX",
  "is_ambiguous": false,
  "confidence": "high"
}
```

## Formatting Rules

- If `declaration_elements` is an empty list, write `"N/A"` in the section.
- If `hs6_code` is an empty string, omit the "Primary Candidate" line.
- Always include the GIR Analysis section even if it contains only one entry.
- Omit the "Reason" line entirely when `is_ambiguous = false`.
